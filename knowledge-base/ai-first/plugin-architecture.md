# AI First — Плагинная архитектура

**Источник:** https://youtu.be/yWAJ6ICJL4c

## Принцип

Микроядро + плагины. Каждый элемент системы организован как подключаемый модуль.

## Примеры плагинов

- **Провайдеры** — Telegram, почта, календарь (наращиваются)
- **Артефакты** — in-memory, статические файлы, векторное хранилище (разные реализации)
- **Сессии** — механика ядра, обогащаемая плагинами
- **Tools/Skills/MCP** — подключаемые инструменты

## Преимущества

- Простота — каждый плагин отвечает за свою зону
- Расширяемость — новые провайдеры, хранилища, инструменты добавляются без изменения ядра
- Разделение обязанностей — чёткие границы между слоями
- Безопасность — публичные/приватные сессии как архитектурный принцип

## Почему работает

Эффективность достигается за счёт **чёткого формирования контекста** (context engineering) на каждом слое.
