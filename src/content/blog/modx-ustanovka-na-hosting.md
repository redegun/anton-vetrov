# Установка MODX Revolution на хостинг: пошаговая инструкция

Установка MODX Revolution — процесс несложный, но требующий внимания к деталям. За годы работы я установил MODX на десятки разных хостингов и знаю все подводные камни. В этой инструкции покажу, как правильно установить MODX с первого раза.

## Шаг 1: Выбор и подготовка хостинга

### Системные требования MODX Revolution

Перед установкой убедитесь, что хостинг поддерживает:

- **PHP:** 7.4+ (рекомендую 8.1+)
- **MySQL:** 5.7+ или MariaDB 10.2+
- **Apache/Nginx** с mod_rewrite
- **Память:** минимум 64MB, рекомендую 128MB+
- **Место:** минимум 100MB свободного места

### Рекомендуемый хостинг

Для MODX рекомендую **Timeweb Cloud** — проверенный годами хостинг с отличной поддержкой PHP и MySQL:

**Почему именно Timeweb:**
- Быстрые SSD-диски
- Автоматические резервные копии
- Бесплатный SSL-сертификат
- Техподдержка 24/7
- Удобная панель управления

**[Регистрируйтесь в Timeweb Cloud](https://timeweb.cloud/?i=33633)** — получите бонусы для новых клиентов.

### Создание базы данных

В панели управления хостингом:

1. Перейдите в раздел **«Базы данных MySQL»**
2. Создайте новую базу данных
3. Создайте пользователя БД
4. Назначьте пользователю все права на базу

**Запишите данные:**
- Имя базы данных
- Имя пользователя
- Пароль
- Хост (обычно `localhost`)

```sql
-- Пример создания БД через консоль
CREATE DATABASE modx_site CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'modx_user'@'localhost' IDENTIFIED BY 'strong_password';
GRANT ALL PRIVILEGES ON modx_site.* TO 'modx_user'@'localhost';
FLUSH PRIVILEGES;
```

## Шаг 2: Скачивание MODX Revolution

### Официальный сайт

1. Идите на [modx.com](https://modx.com/download/)
2. Скачайте **MODX Revolution** (последняя стабильная версия)
3. Выберите **Traditional** версию для новичков

```

```

## Шаг 3: Загрузка файлов на сервер

### Через FTP

Используйте любой FTP-клиент (FileZilla, WinSCP):

1. Подключитесь к серверу по FTP
2. Распакуйте архив MODX локально
3. Загрузите все файлы в корень сайта
4. Убедитесь, что права доступа корректны

### Через файл-менеджер

Многие хостинги предоставляют веб-файлменеджер:

1. Загрузите ZIP-архив в корень сайта
2. Распакуйте через файлменеджер
3. Переместите файлы из подпапки в корень (если нужно)

### Правильная структура папок

```
public_html/            # Корень сайта
├── setup/
├── core/
├── manager/
├── assets/
├── connectors/
├── index.php
└── .htaccess           # Важно для ЧПУ!
```

## Шаг 4: Настройка прав доступа

### Критически важные права

Установите корректные права через FTP или SSH:

```bash
# Папки - 755
chmod 755 assets/
chmod 755 core/cache/
chmod 755 core/components/
chmod 755 core/export/
chmod 755 core/packages/
chmod 755 core/import/

# core/config/ должна быть записываемой
chmod 775 core/config/

# Файлы - 644
chmod 644 index.php
chmod 644 .htaccess
```

### Через панель управления

В большинстве панелей управления:

1. Выберите папку/файл
2. Нажмите **«Права доступа»** или **«Chmod»**
3. Установите нужные значения

## Шаг 5: Запуск установки

### Переход к мастеру установки

1. Откройте `ваш-домен.ru/setup/` в браузере
2. Выберите язык установки (Russian)
3. Нажмите **«Далее»**

### Проверка системы

Мастер проверит:
- Версию PHP
- Доступность расширений
- Права доступа к папкам
- Наличие mod_rewrite

**Типичные ошибки и решения:**

```
❌ Папка core/cache недоступна для записи
✅ Решение: chmod 775 core/cache/

❌ Расширение PDO не найдено
✅ Решение: включите PDO в настройках хостинга

❌ mod_rewrite недоступен
✅ Решение: обратитесь в поддержку хостинга
```

## Шаг 6: Настройка базы данных

### Ввод данных подключения

На экране **«База данных»** введите:

- **Хост:** `localhost` (или IP от хостера)
- **Имя БД:** созданная ранее база
- **Пользователь:** пользователь БД
- **Пароль:** пароль пользователя
- **Префикс таблиц:** `modx_` (по умолчанию)

### Проверка соединения

Нажмите **«Тест подключения»** — должно появиться ✅ «Соединение установлено».

### Создание таблиц

MODX создаст структуру базы автоматически:

```sql
-- Основные таблицы MODX
modx_site_content       -- Ресурсы (страницы)
modx_site_templates     -- Шаблоны
modx_site_snippets      -- Сниппеты  
modx_site_chunks        -- Чанки
modx_site_tmplvars      -- TV-поля
modx_users              -- Пользователи
-- И еще ~50 системных таблиц
```

## Шаг 7: Создание администратора

### Данные суперпользователя

Введите данные главного администратора:

- **Логин:** выберите уникальный (не admin!)
- **Email:** рабочий email для восстановления
- **Пароль:** сложный пароль (12+ символов)

```
Хороший пример:
Логин: site_admin_2026
Email: admin@вашдомен.ru
Пароль: Modx!Secure#2026
```

### Контекст и настройки

- **Контекст:** `web` (основной сайт)
- **Часовой пояс:** `Europe/Moscow`
- **Кодировка:** `UTF-8`

## Шаг 9: Установка и первая проверка

### Процесс установки

1. Нажмите **«Установить»**
2. Дождитесь завершения (1-3 минуты)
3. Увидите **«Установка завершена успешно»**

### Первые действия

**Обязательно сразу после установки:**

1. **Удалите папку setup/** — критично для безопасности!
```bash
rm -rf setup/
```

2. **Проверьте главную страницу:** `ваш-домен.ru`
3. **Войдите в админку:** `ваш-домен.ru/manager/`

### Тест основных функций

```php
// Проверьте, что работает:
- Главная страница загружается
- Админка открывается
- Можно создать тестовую страницу
- ЧПУ работают (проверьте в настройках)
```

## Шаг 10: Начальная настройка

### Включение ЧПУ (SEO URL)

1. **Система** → **Настройки системы**
2. Найдите **`friendly_urls`** → установите **«Да»**
3. **`use_alias_path`** → **«Да»**
4. Сохраните и очистите кэш

### Проверка .htaccess

Убедитесь, что файл `.htaccess` в корне сайта содержит:

```apache
RewriteEngine On
RewriteBase /

# Исключаем файлы и папки
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d

# Перенаправляем на index.php
RewriteRule ^(.*)$ index.php?q=$1 [L,QSA]
```

### Настройка безопасности

Измените стандартные пути (для безопасности):

```php
// core/config/config.inc.php
$modx_core_path = '/home/user/modx_core/';  // Вне public_html
$modx_processors_path = $modx_core_path . 'model/modx/processors/';
$modx_connectors_path = $modx_core_path . 'connectors/';
```

## Устранение типичных проблем

### Ошибка 500 после установки

**Причины и решения:**

```php
// 1. Неправильные права доступа
chmod 644 .htaccess
chmod 755 core/cache/

// 2. Проблемы с mod_rewrite
# Временно отключите в .htaccess:
# RewriteEngine Off

// 3. Ошибки в конфиге
# Проверьте core/config/config.inc.php
```

### Админка не открывается

```php
// Проверьте:
1. Правильность URL: domain.ru/manager/
2. Файл manager/index.php существует
3. Права доступа корректны
4. База данных доступна
```

### Страницы не открываются

```bash
# Проверьте настройки ЧПУ:
1. friendly_urls = Да
2. use_alias_path = Да  
3. .htaccess существует и читается
4. mod_rewrite включен на сервере
```
