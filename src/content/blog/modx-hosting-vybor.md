## Требования MODX к хостингу

### Минимальные системные требования

MODX Revolution достаточно требователен к серверу:

```
PHP: 7.4+ (рекомендую 8.1+)
MySQL: 5.7+ или MariaDB 10.2+
Память: 128MB+ (рекомендую 256MB+)
Место: 200MB+ для базовой установки
Apache/Nginx с mod_rewrite
OpenSSL для безопасности
```

## Типы хостинга для MODX

### Shared-хостинг (виртуальный)

**Кому подходит:** Малый бизнес, простые сайты

**Плюсы:**
- Дешёво (100-500₽/мес)  
- Не нужно администрировать
- Быстрый запуск

**Минусы:**
- Соседи по серверу могут влиять на скорость
- Ограничения ресурсов
- Нет root-доступа

**Пример проекта:** Сайт компании на 10-20 страниц

```php
// Настройки MODX для shared-хостинга
cache_enabled = 1
cache_db = 1  
cache_resource_expires = 86400 // Кэш на сутки
```

### VPS/VDS (виртуальный сервер)

**Кому подходит:** Средний бизнес, нагруженные сайты

**Плюсы:**
- Гарантированные ресурсы
- Можно настроить под MODX  
- Root-доступ
- Предсказуемая производительность

**Минусы:**
- Нужны знания администрирования
- Дороже (от 500₽/мес)

**Пример проекта:** Интернет-магазин, корпоративный портал

### Выделенный сервер

**Кому подходит:** Крупные проекты, высокие нагрузки

**Плюсы:**
- Максимальная производительность
- Полный контроль
- Можно оптимизировать под MODX

**Минусы:**
- Дорого (от 5000₽/мес)
- Нужен системный администратор


## Настройка хостинга под MODX

### Оптимизация PHP

Попросите хостер установить:

```ini
; Оптимальные настройки для MODX
memory_limit = 256M
max_execution_time = 120
max_input_vars = 10000
upload_max_filesize = 64M
post_max_size = 64M
max_file_uploads = 50

; OPCache для ускорения
opcache.enable = 1
opcache.memory_consumption = 128
opcache.revalidate_freq = 60
```

### Оптимизация MySQL

```sql
-- Настройки MySQL для MODX
[mysqld]
innodb_buffer_pool_size = 128M
query_cache_size = 32M  
query_cache_type = 1
tmp_table_size = 32M
max_heap_table_size = 32M
```

### Настройка .htaccess

```apache
# Оптимизированный .htaccess для MODX
RewriteEngine On

# Сжатие статики
<IfModule mod_deflate.c>
    AddOutputFilterByType DEFLATE text/html text/css text/javascript application/javascript text/xml
</IfModule>

# Кэширование браузера
<IfModule mod_expires.c>
    ExpiresActive On
    ExpiresByType text/css "access plus 1 month"
    ExpiresByType application/javascript "access plus 1 month"
    ExpiresByType image/png "access plus 1 year"
    ExpiresByType image/jpg "access plus 1 year"
    ExpiresByType image/jpeg "access plus 1 year"
    ExpiresByType image/gif "access plus 1 year"
</IfModule>

# Основные правила MODX
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ index.php?q=$1 [L,QSA]
```

## Миграция MODX между хостингами

### Подготовка к переносу

1. **Сделайте полный бэкап:**

```bash
# Архив файлов
tar -czf site-backup.tar.gz public_html/

# Дамп базы данных  
mysqldump -u user -p database_name > site-backup.sql
```

2. **Проверьте системные требования** нового хостинга

### Процесс миграции

1. **Загрузите файлы** на новый хостинг
2. **Создайте базу данных** и импортируйте дамп
3. **Обновите конфиги:**

```php
// core/config/config.inc.php
$database_server = 'новый_хост';
$database_user = 'новый_пользователь';  
$database_password = 'новый_пароль';
$database_connection_charset = 'utf8mb4';

// Обновите пути если нужно
$modx_core_path = '/новый/путь/core/';
```

4. **Очистите кэш**
5. **Проверьте работоспособность**

### Типичные проблемы при миграции

```php
// Проблема: Сайт не открывается
// Решение: Проверьте настройки БД в config.inc.php

// Проблема: 500 ошибка  
// Решение: Проверьте права доступа (755 для папок, 644 для файлов)

// Проблема: Админка не работает
// Решение: Очистите кэш, проверьте пути в конфиге
```

## Специфика MODX: на что обратить внимание

### Административная панель

MODX использует ExtJS — тяжёлую JavaScript-библиотеку. Нужен хостинг с:
- Быстрым процессором  
- Достаточной оперативной памятью
- Хорошим интернет-каналом

### Кэширование

MODX активно использует файловый кэш:
- Нужны быстрые диски (SSD)
- Права на запись в папку `core/cache/`
- Достаточно места для кэш-файлов

### База данных

MODX создаёт много таблиц (~70 по умолчанию):
- MySQL должен быть версии 5.7+
- Нужны права на создание таблиц и индексов
- Рекомендую InnoDB как движок БД

## Частые вопросы

**Q: Можно ли использовать бесплатный хостинг для MODX?**
A: Технически можно, но не рекомендую. Ограничения по ресурсам делают MODX медленным, а отсутствие поддержки создаёт проблемы.

**Q: Нужен ли VPS для простого сайта на MODX?**  
A: Для большинства проектов хватает качественного shared-хостинга. VPS нужен при > 1000 посетителей/день.

**Q: Влияет ли местоположение сервера на MODX?**
A: Да, для российской аудитории выбирайте хостинг с серверами в России. Это ускоряет загрузку на 100-300ms.

**Q: Можно ли перенести MODX с одного хостинга на другой?**
A: Да, миграция MODX достаточно простая. Главное — сохранить структуру файлов и правильно настроить базу данных.
