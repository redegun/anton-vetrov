# Топ-20 дополнений MODX Revolution: must-have для каждого сайта

За 20 лет работы с MODX я перепробовал сотни дополнений. Многие из них оказались бесполезными, но некоторые стали незаменимыми инструментами в моем арсенале. Сегодня поделюсь топом дополнений, которые использую практически в каждом проекте.

## Что такое дополнения MODX

Дополнения (packages) — это расширения, которые добавляют новую функциональность в MODX. Они могут включать:
- Сниппеты для вывода контента
- Плагины для автоматизации
- Компоненты управления (CMP)
- Виджеты для дашборда
- TV-поля и input-типы

Все дополнения устанавливаются через **Управление пакетами** в админке MODX или скачиваются с [modstore.pro](https://modstore.pro/).

## Категория: Основа каждого сайта

### 1. pdoTools — ускоритель MODX

**Что делает:** Набор быстрых сниппетов для работы с базой данных.

**Зачем нужно:** 
- В 5-10 раз быстрее стандартных сниппетов MODX
- Единый синтаксис для всех задач
- Встроенный шаблонизатор Fenom

**Основные сниппеты:**
- `pdoMenu` — меню любой сложности
- `pdoResources` — вывод списка ресурсов
- `pdoUsers` — работа с пользователями
- `pdoPage` — пагинация

**Пример использования:**
```html
<!-- Вывод новостей с пагинацией -->
[[pdoPage?
    &element=`pdoResources`
    &parents=`5`
    &limit=`10`
    &tpl=`newsItem`
    &ajaxMode=`default`
]]

<div class="pagination">
    [[page.nav]]
</div>
```

**Ссылка:** [pdoTools на MODStore](https://modstore.pro/packages/utilities/pdotools)

### 2. FormIt — обработка форм

**Что делает:** Мощный процессор форм с валидацией, защитой от спама и интеграциями.

**Зачем нужно:**
- Обработка любых форм (контакты, заказы, регистрация)
- Встроенная защита от спама
- Email-уведомления и автоответы
- Сохранение в базу данных

**Пример формы обратной связи:**
```html
[[FormIt?
    &hooks=`spam,email,redirect`
    &emailTpl=`contactEmail`
    &emailTo=`info@site.ru`
    &emailSubject=`Новое сообщение с сайта`
    &redirectTo=`[[++thankspage]]`
    &validate=`name:required,email:required:email,message:required`
]]

<form action="[[~[[*id]]]]" method="post">
    <input type="text" name="name" value="[[+fi.name]]" placeholder="Имя" required>
    <span class="error">[[+fi.error.name]]</span>
    
    <input type="email" name="email" value="[[+fi.email]]" placeholder="Email" required>
    <span class="error">[[+fi.error.email]]</span>
    
    <textarea name="message" placeholder="Сообщение" required>[[+fi.message]]</textarea>
    <span class="error">[[+fi.error.message]]</span>
    
    <button type="submit">Отправить</button>
</form>
```

**Ссылка:** [FormIt на MODStore](https://modstore.pro/packages/forms/formit)

### 3. MIGX — кастомные TV-поля

**Что делает:** Создание сложных TV-полей с множественными значениями.

**Зачем нужно:**
- Галереи изображений
- Списки услуг/товаров
- Таблицы характеристик
- Любые повторяющиеся блоки

**Пример использования — галерея:**
```html
<!-- В ресурсе TV-поле gallery с MIGX -->
[[getImageList?
    &tvname=`gallery`
    &tpl=`galleryItem`
]]

<!-- Чанк galleryItem -->
<div class="gallery-item">
    <img src="[[+image]]" alt="[[+title]]">
    <p>[[+description]]</p>
</div>
```

**Ссылка:** [MIGX на MODStore](https://modstore.pro/packages/utilities/migx)

## Категория: Электронная коммерция

### 4. miniShop2 — интернет-магазин

**Что делает:** Полнофункциональный интернет-магазин для MODX.

**Зачем нужно:**
- Каталог товаров с фильтрацией
- Корзина и оформление заказов
- Система скидок и купонов
- Интеграция с платёжными системами

**Возможности:**
- Неограниченное количество товаров и категорий
- Множественные изображения товаров
- Варианты товаров (размер, цвет)
- Система отзывов и рейтингов
- CRM для управления заказами

**Ссылка:** [miniShop2 на MODStore](https://modstore.pro/packages/ecommerce/minishop2)

### 5. mFilter2 — фильтрация товаров

**Что делает:** Аjax-фильтрация для каталогов товаров.

**Зачем нужно:**
- Фильтры по любым характеристикам
- Быстрая фильтрация без перезагрузки страницы
- SEO-дружелюбные URL для фильтров

**Пример фильтров:**
```html
[[mFilter2]]
<!-- Автоматически генерирует фильтры для товаров -->

<div id="mse2_mfilter">
    <!-- Здесь будут результаты фильтрации -->
    [[mSearch2?
        &parents=`5`
        &tpl=`productCard`
        &limit=`20`
        &ajaxMode=`default`
    ]]
</div>
```

**Ссылка:** [mFilter2 на MODStore](https://modstore.pro/packages/ecommerce/mfilter2)

## Категория: SEO и оптимизация

### 9. StercSEO — комплексное SEO

**Что делает:** Полный набор инструментов для поисковой оптимизации.

**Зачем нужно:**

**Основные функции:**

- Канонические URL
- Мониторинг 404 ошибок
- Массовое редактирование SEO-тегов

**Ссылка:** [StercSEO на MODStore](https://modstore.pro/packages/seo/stercseo)

### 10. Sitemap — карты сайта

**Что делает:** Автоматическая генерация XML и HTML карт сайта.

**Ссылка:** [Sitemap на MODStore](https://modstore.pro/packages/seo/sitemap)

### 11. SimpleSearch — поиск по сайту

**Что делает:** Простой и быстрый поиск по содержимому сайта.

## Как выбрать дополнения для проекта

### Корпоративный сайт (базовый набор):
1. **pdoTools** — основа
2. **FormIt** — формы обратной связи  
3. **StercSEO** — поисковая оптимизация
4. **Ace** — удобство разработки
5. **MIGX** — гибкие поля

### Интернет-магазин:
1. **pdoTools** — основа
2. **miniShop2** — каталог и заказы
3. **mFilter2** — фильтрация товаров
4. **StercSEO** — продвижение
5. **FormIt** — дополнительные формы

### Новостной портал:
1. **pdoTools** — основа
2. **Articles** или **Tickets** — блог-функции
3. **SimpleSearch** — поиск по сайту
4. **StercSEO** — продвижение
5. **sendex** — email-подписки

## Установка дополнений

### Через MODStore (рекомендуется):
1. Идём в **Система** → **Управление пакетами**
2. Вкладка **Скачать дополнения**
3. Ищем нужное дополнение
4. Нажимаем **Скачать** → **Установить**

### Ручная установка:
1. Скачиваем .transport.zip
2. **Управление пакетами** → **Загрузить пакет**
3. Выбираем файл → **Загрузить**
4. Устанавливаем пакет

## Лучшие практики использования

### 1. Не устанавливайте лишнего
Каждое дополнение — это дополнительная нагрузка. Используйте только то, что реально нужно.

### 2. Обновляйте регулярно  
Следите за обновлениями дополнений. Они часто содержат исправления безопасности.

### 3. Тестируйте на dev-среде
Новые дополнения сначала тестируйте на тестовом сайте.

### 4. Читайте документацию
У каждого дополнения есть документация на modx.pro или в самом пакете.

## Заключение

Правильно подобранные дополнения превращают MODX в мощный инструмент для решения любых задач. Мой список прошёл проверку временем — каждое дополнение использовалось в десятках реальных проектов.

Начните с базовых дополнений (pdoTools, FormIt, MIGX), а затем добавляйте специализированные под конкретные задачи. И помните: качество важнее количества.
