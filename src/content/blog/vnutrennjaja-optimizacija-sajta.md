Внутренняя оптимизация — это работа над самим сайтом: контентом, структурой, мета-тегами и перелинковкой. Это то, что вы контролируете на 100%, в отличие от внешних ссылок.

Ниже — полный чек-лист. Пройдитесь по каждому пункту и отметьте, что уже сделано.

## Мета-теги

Мета-теги — первое, что видит поисковик и пользователь в выдаче.

### Title

Title — самый важный мета-тег. Он отображается как заголовок сниппета в выдаче.

**Правила:**
- Длина: 50–60 символов (длинные обрезаются)
- Содержит основной ключевой запрос
- Уникальный для каждой страницы
- Читается как нормальное предложение, а не набор ключей
- Начинается с ключевого слова (если это естественно)

**Плохо:** `SEO оптимизация, продвижение сайтов, раскрутка — купить SEO`

**Хорошо:** `SEO-оптимизация сайта: полное руководство на 2025 год`

### Description

Description — описание страницы в сниппете. Не влияет на ранжирование напрямую, но влияет на CTR (кликабельность).

**Правила:**
- Длина: 120–160 символов
- Содержит ключевое слово (выделяется жирным в выдаче)
- Мотивирует кликнуть (конкретика, выгода)
- Уникальный для каждой страницы

### H1

H1 — основной заголовок на странице. Должен быть один.

**Правила:**
- Один H1 на страницу
- Содержит основной ключ
- Не дублирует title (перефразируйте)
- Первое, что видит пользователь

Подробный гайд: [Meta-теги: title, description, h1 — как правильно](/blog/meta-tegi-title-description-h1/).

## Структура заголовков (H1–H6)

Заголовки создают иерархию контента. Поисковик использует их для понимания структуры страницы.

```
H1: Основная тема
  H2: Подтема 1
    H3: Детали подтемы 1
  H2: Подтема 2
    H3: Детали подтемы 2
    H3: Ещё детали
  H2: Подтема 3
```

**Правила:**
- Не пропускайте уровни (H1 → H3 без H2 — ошибка)
- Используйте ключевые слова в H2 и H3, но естественно
- Каждый H2 — отдельный смысловой блок
- H2 и H3 помогают попасть в «быстрые ответы» Яндекса

## Контент

### Требования к тексту

| Параметр | Рекомендация |
|----------|-------------|
| Уникальность | > 90% (проверять text.ru, Advego) |
| Водность | < 60% |
| Тошнота (академическая) | < 8% |
| Длина | Зависит от темы (смотрите конкурентов) |
| Структура | H2, H3, списки, таблицы, изображения |

### Полнота ответа

Страница должна полностью отвечать на запрос пользователя. Проверьте:

1. Откройте топ-5 выдачи по целевому запросу
2. Выпишите все подтемы, которые освещают конкуренты
3. Убедитесь, что ваш контент раскрывает всё то же + что-то дополнительное
4. Добавьте уникальный контент: опыт, данные, примеры, кейсы

### E-E-A-T

Google оценивает контент по критериям E-E-A-T:

- **Experience** — есть ли у автора реальный опыт?
- **Expertise** — является ли автор экспертом?
- **Authoritativeness** — авторитетен ли источник?
- **Trustworthiness** — можно ли доверять информации?

**Как показать E-E-A-T:**
- Укажите автора с биографией
- Ссылайтесь на источники
- Приводите собственный опыт и кейсы
- Обновляйте контент

Подробнее о написании текстов: [SEO-оптимизация текста: как писать для поисковиков](/blog/seo-optimizacija-teksta/).

## URL-адреса

**Правила:**

- Короткие и понятные: `/blog/seo-audit-sajta/` вместо `/blog/post123/`
- На латинице (транслитерация)
- Содержат ключевое слово
- Используют дефисы, не подчёркивания
- Без параметров и цифровых ID
- Не меняйте URL после индексации (если нужно — сделайте 301 редирект)

## Изображения

Изображения влияют на SEO через:

- **alt-текст** — описание изображения для поисковика и скринридеров
- **Имя файла** — `seo-audit-checklist.webp` лучше, чем `IMG_2847.jpg`
- **Размер** — оптимизируйте вес (WebP, сжатие)
- **Lazy loading** — отложенная загрузка для скорости

**Чек-лист для изображений:**
- [ ] Каждое изображение имеет alt-текст с описанием
- [ ] Файлы в формате WebP
- [ ] Размер файла < 100 КБ (для hero — до 200 КБ)
- [ ] Размеры (width, height) указаны в HTML
- [ ] Используется lazy loading для изображений ниже первого экрана

## Внутренняя перелинковка

Перелинковка — связывание страниц сайта между собой ссылками. Это помогает:

- Распределить «ссылочный вес» между страницами
- Помочь поисковому роботу найти все страницы
- Удерживать пользователя на сайте

**Правила перелинковки:**

1. Ссылайтесь из контента, а не из сайдбара / футера
2. Используйте анкоры с ключевыми словами (но разнообразные)
3. Ссылайтесь на тематически связанные страницы
4. Важные страницы должны получать больше внутренних ссылок
5. Не переусердствуйте — 3–5 внутренних ссылок на 1000 слов

**Модель «кокона»:**
- Группируйте страницы по темам
- Внутри группы — плотная перелинковка
- Между группами — ссылки только на уровне главных страниц

## Скорость загрузки

Скорость — фактор ранжирования и в Яндексе, и в Google.

**Как ускорить:**
- Оптимизировать изображения (WebP, сжатие)
- Включить кеширование
- Минифицировать CSS и JavaScript
- Использовать CDN
- Выбрать быстрый хостинг — например, [Timeweb](https://timeweb.com/ru/?i=33633)

**Цель:** < 3 секунд на мобильном, > 90 баллов в PageSpeed Insights.

## Мобильная оптимизация

Более 60% трафика — с мобильных устройств. Яндекс и Google используют mobile-first indexing.

**Чек-лист:**
- [ ] Адаптивная вёрстка (responsive design)
- [ ] Шрифт читается без увеличения (минимум 16px)
- [ ] Кнопки достаточно большие для нажатия пальцем
- [ ] Нет горизонтальной прокрутки
- [ ] Формы удобны на мобильном

## Микроразметка (Schema.org)

Структурированные данные помогают поисковикам лучше понять контент и показывать расширенные сниппеты.

**Основные типы:**
- **Article** — для статей блога
- **FAQ** — для страниц с вопросами и ответами
- **Breadcrumb** — хлебные крошки
- **Organization** — информация о компании
- **Product** — для товаров
- **HowTo** — для инструкций

**Как проверить:** Google Rich Results Test, Яндекс.Вебмастер → Валидатор микроразметки.
