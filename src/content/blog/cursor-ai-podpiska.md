Cursor — это не просто «ещё одна нейросеть». Это полноценная IDE (редактор кода), построенная как форк VS Code, но с глубоко интегрированным AI. Если ChatGPT — это чат, в который можно вставлять код, то Cursor — это редактор, в котором AI понимает весь ваш проект.

## Что такое Cursor и зачем он нужен

Cursor выглядит и работает как VS Code — те же расширения, горячие клавиши, настройки. Но внутри — AI-движок, который:

- **Видит весь проект** — не только открытый файл, а всю кодовую базу
- **Предсказывает код** — автокомплит на уровне целых блоков, а не отдельных слов
- **Редактирует по инструкции** — описываешь, что нужно изменить, Cursor делает
- **Работает с несколькими файлами** — может рефакторить сразу несколько файлов за один промпт

Это не плагин. Это IDE, где AI — первоклассный инструмент.

## Ключевые возможности

### Tab — предиктивный автокомплит

Самая используемая функция. Cursor анализирует контекст и предлагает продолжение кода. Не одну строку, а целые блоки — функции, условия, циклы.

Нажимаешь **Tab** — код вставляется. Не нравится — пишешь дальше, Cursor подстроится.

### Cmd+K — инлайн-редактирование

Выделяешь участок кода, нажимаешь Cmd+K, пишешь инструкцию:

> "Добавь обработку ошибок и логирование"

Cursor переписывает выделенный код по твоей инструкции. Показывает diff — ты решаешь, принять или нет.

### Composer — мультифайловое редактирование

Самая мощная функция Cursor Pro. Описываешь задачу:

> "Создай API endpoint для загрузки файлов. Нужен маршрут, контроллер, валидация и тесты."

Cursor создаёт или редактирует **несколько файлов одновременно**. Видишь все изменения, принимаешь по одному или все сразу.

### Chat — контекстный чат

Как ChatGPT, но с доступом к вашему проекту. Можно спросить:
- "Как работает функция X в этом проекте?"
- "Найди баги в модуле авторизации"
- "Объясни этот legacy-код"

Chat знает о файлах проекта, зависимостях, конфигурации.

### .cursorrules — кастомизация поведения

Файл `.cursorrules` в корне проекта задаёт правила для AI:
- Стиль кода (naming conventions, форматирование)
- Используемые технологии
- Запреты (не использовать определённые паттерны)

## Тарифы Cursor: Hobby vs Pro vs Business

| Параметр | Hobby (бесплатно) | Pro ($20/мес) | Business ($40/мес) |
|----------|-------------------|---------------|---------------------|
| Быстрые запросы | 50 | 500 | 500 |
| Медленные запросы | 200 | Безлимит | Безлимит |
| Модели | GPT-4o mini | GPT-4o, Claude Sonnet | GPT-4o, Claude Sonnet |
| Composer | ❌ | ✅ | ✅ |
| Tab (автокомплит) | Ограничено | ✅ | ✅ |
| Приватность (zero retention) | ❌ | ❌ | ✅ |
| Админ-панель команды | ❌ | ❌ | ✅ |

**Быстрые запросы** — ответ за 5–15 секунд (GPT-4o / Claude Sonnet).
**Медленные запросы** — в очереди, ответ за 30–120 секунд (когда быстрые закончились).

## Cursor Pro — что реально получаешь за $20

500 быстрых запросов в месяц — это примерно **16–17 запросов в день**. Для активной разработки может быть мало, но медленные запросы — безлимит.

На практике:
- Tab (автокомплит) работает **бесплатно** и безлимитно — это основная рабочая функция
- Быстрые запросы тратятся на Composer и Chat
- Медленные запросы — те же модели, просто дольше ждать

Для большинства разработчиков **Pro хватает**. Business нужен только для команд.

## Мой опыт с Cursor Pro

Перешёл на Cursor с VS Code + Copilot. Разница заметна:

- **Composer** — экономит часы на рефакторинге. Описал задачу — получил изменения в 5 файлах.
- **Tab** — автокомплит реально предугадывает, что ты собираешься написать. Не отдельные слова, а целые блоки логики.
- **Контекст проекта** — Chat знает о структуре проекта, что избавляет от копирования кода в ChatGPT.

Из минусов — 500 быстрых запросов заканчиваются к концу месяца, если активно использовать Composer. Но медленные запросы спасают.

**Вердикт:** для профессионального разработчика Cursor Pro — must have. $20/мес окупаются за первый день.
