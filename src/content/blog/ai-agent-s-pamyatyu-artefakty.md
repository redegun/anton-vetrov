Обычный чат-бот забывает всё после окончания разговора. В лучшем случае — помнит текущую сессию. Это как сотрудник, который каждое утро приходит на работу и не помнит, что делал вчера.

В [AI First архитектуре](/blog/arhitektura-ai-sistem-4-sloya/) третий слой — артефакты. Это долгосрочная память системы: результаты предыдущей работы, которые сохраняются и используются в будущем. Именно артефакты превращают чат-бота в полноценного [цифрового работника](/blog/cifrovye-rabotniki-ai-agenty/).

## Что такое артефакт

Артефакт — это результат работы AI-системы, зафиксированный для повторного использования. Типы артефактов:

- **Заметки** — краткие записи о клиентах, сделках, событиях
- **Планы** — стратегии, планы действий, чек-листы
- **Отчёты** — ежедневные сводки, анализы, прогнозы
- **Документы** — черновики КП, договоров, писем
- **Профили** — накопленные знания о клиентах, конкурентах, рынке

Артефакты могут храниться как статические MD-файлы (простой и надёжный формат) или в векторном хранилище (для больших объёмов с семантическим поиском). Тип хранения — это [плагин](/blog/plaginnaya-arhitektura-ai/), его можно менять без переделки системы.

## Двойная роль артефактов

У артефактов два назначения, и именно это делает их мощным инструментом:

### 1. Фиксация результатов

LLM выполнила задачу — результат сохраняется как артефакт. Агент-аналитик провёл анализ продаж → артефакт «Отчёт за 14 марта». Агент продаж квалифицировал лид → артефакт «Профиль клиента X».

Без фиксации результаты теряются. С фиксацией — система накапливает знания.

### 2. Подмешивание в контекст

Артефакт становится частью динамического контекста при следующих запросах. Это вторая часть [контекст-инжиниринга](/blog/kontekst-inzhiniring-vazhneje-vybora-modeli/): системный промпт (статика) + **динамический контекст (артефакты, данные)** + промпт (текущий запрос).

Агент продаж квалифицирует нового лида. В контекст подмешивается:
- Вчерашний отчёт по продажам (артефакт агента-аналитика)
- Профиль похожего клиента из прошлого месяца (артефакт)
- Текущие приоритеты компании (артефакт руководства)

Агент не просто отвечает на запрос — он отвечает с учётом всего накопленного опыта.

## Артефакты как мост между агентами

В системе с несколькими [цифровыми работниками](/blog/cifrovye-rabotniki-ai-agenty/) артефакты — это основной способ обмена знаниями. Агенты не общаются напрямую — они общаются через артефакты.

### Пример рабочего дня

**06:00** — Крон запускает агента-аналитика. Он собирает данные из CRM, анализирует вчерашние продажи, создаёт артефакт «Ежедневный отчёт: 14 марта».

**09:15** — Приходит заявка в Telegram. [Провайдер](/blog/provajdery-i-klienty-ai-sistemy/) передаёт промпт агенту продаж. Агент подгружает утренний отчёт (артефакт аналитика) — знает текущую ситуацию. Квалифицирует лид, создаёт артефакт «Профиль: Компания Y».

**11:30** — Менеджер просит подготовить КП для компании Y. Агент-документалист подгружает профиль компании (артефакт агента продаж) и шаблон КП (артефакт-шаблон). Генерирует персонализированный документ.

**17:00** — Агент-аналитик подводит итоги дня. Подгружает все созданные за день артефакты, формирует сводку. Она станет контекстом для завтрашнего утра.

Каждый агент работает в своей зоне, но все связаны через артефакты. Это принцип разделения труда в действии — тот самый, который Адам Смит описал для булавочной фабрики.

## Типы хранения артефактов

### Статические MD-файлы

Самый простой и надёжный вариант. Артефакт — это Markdown-файл с метаданными:

```markdown
---
type: daily-report
agent: analyst
date: 2026-03-14
---

# Ежедневный отчёт: 14 марта

## Продажи
- Новых лидов: 12 (+3 к среду)
- Квалифицированных: 7 (58%)
- Закрытых сделок: 2 (средний чек: 150 000 ₽)

## Тренды
- Растёт спрос на AI-интеграцию с CRM
- 40% лидов приходят из Telegram
```

Плюсы: просто, читаемо, версионируемо (Git). Минусы: не масштабируется на тысячи документов с семантическим поиском.

### Векторное хранилище

Для больших объёмов: артефакты разбиваются на чанки, эмбеддятся и хранятся в векторной базе. При запросе система находит семантически релевантные фрагменты.

Плюсы: масштабируется, семантический поиск. Минусы: сложнее в настройке, менее прозрачно.

### Гибридный подход

MD-файлы для структурированных артефактов (отчёты, планы) + векторное хранилище для базы знаний (документация, FAQ, истории клиентов). Оба варианта — [плагины](/blog/plaginnaya-arhitektura-ai/) в архитектуре, работают параллельно.

## Управление артефактами

### Жизненный цикл

1. **Создание** — агент выполняет задачу и фиксирует результат
2. **Использование** — артефакт подмешивается в контекст других запросов
3. **Обновление** — при новых данных артефакт обновляется (профиль клиента)
4. **Архивирование** — устаревшие артефакты перемещаются в архив
5. **Удаление** — когда данные больше не нужны

### Релевантность

Не все артефакты нужны для каждого запроса. Система выбирает релевантные на основе:
- Типа задачи — квалификация лида требует профили и отчёты, а не шаблоны документов
- Давности — вчерашний отчёт релевантнее прошлогоднего
- Связей — артефакты, созданные для этого клиента, приоритетнее общих

Это часть [контекст-инжиниринга](/blog/kontekst-inzhiniring-vazhneje-vybora-modeli/) — не просто подмешивать всё подряд, а выбирать то, что действительно нужно.

### Безопасность

Артефакты наследуют уровень доступа сессии, в которой созданы. Артефакт из [приватной сессии](/blog/bezopasnost-ai-agentov-sessii/) недоступен в публичной. Агент поддержки на сайте не видит внутренние отчёты по продажам.

## Чем артефакты отличаются от RAG

RAG (Retrieval-Augmented Generation) — это подход, когда система ищет релевантные документы перед генерацией ответа. Артефакты шире:

- **RAG** — поиск в существующей базе знаний
- **Артефакты** — создание новых знаний в процессе работы + их использование

Артефакты — это RAG + генерация. Система не только ищет существующие знания, но и создаёт новые, которые затем используются для обогащения контекста.

## Как начать работать с артефактами

1. **Определите типы** — какие артефакты создают ваши процессы? Отчёты, профили, документы?
2. **Выберите хранение** — MD-файлы для начала, векторное хранилище по мере роста
3. **Настройте создание** — какой агент что создаёт, по какому триггеру
4. **Настройте подмешивание** — какие артефакты релевантны для каких задач
5. **Настройте жизненный цикл** — обновление, архивирование, удаление

Или закажите [проектирование AI-архитектуры](/services/ai-arhitektura/) — спроектирую систему артефактов под ваши процессы.

## Итог

Артефакты — это долгосрочная память AI-системы. Двойная роль: фиксация результатов и обогащение контекста. Мост между [агентами](/blog/cifrovye-rabotniki-ai-agenty/) — они общаются через знания, а не напрямую.

Без артефактов AI-агент — это чат-бот, который каждый день начинает с нуля. С артефактами — цифровой сотрудник, который накапливает опыт и становится полезнее с каждым днём.

Артефакты — третий слой [AI First архитектуры](/blog/arhitektura-ai-sistem-4-sloya/).
