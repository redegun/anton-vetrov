Редизайн — одна из самых рискованных операций для SEO. Неправильно проведённое обновление может обрушить позиции за неделю, а восстановление займёт месяцы. При этом грамотный [редизайн](/services/redizajn-sajta/) не только сохраняет, но и улучшает поисковые показатели.

Разберу по шагам, что нужно сделать до, во время и после обновления сайта.

## Почему позиции падают после редизайна

Главные причины потери трафика после обновления:

- **Изменились URL без редиректов.** Поисковик приходит на старый адрес, получает 404, удаляет страницу из индекса. Весь накопленный авторитет страницы (ссылки, поведенческие метрики) теряется.
- **Пропали мета-теги.** Новый шаблон «забыл» про title, description, canonical. Страницы начинают конкурировать друг с другом.
- **Изменилась структура контента.** Текст, который ранжировался, переместился или исчез. Тематическая релевантность страницы упала.
- **Скорость просела.** Новый дизайн с тяжёлыми анимациями и неоптимизированными изображениями ухудшил Core Web Vitals.

Всё это можно предотвратить. Нужна подготовка.

## Шаг 1. Зафиксируйте текущее состояние

Перед любыми изменениями создайте «снимок» сайта:

**Структура и URL:**
- Выгрузите полный список URL через Screaming Frog, Яндекс.Вебмастер или Google Search Console.
- Для каждого URL зафиксируйте: title, description, H1, canonical, статус-код.

**Позиции:**
- Снимите позиции по ключевым запросам. Это ваш benchmark для сравнения после запуска.

**Трафик:**
- Сохраните данные из Яндекс.Метрики / Google Analytics за последние 3 месяца: страницы входа, конверсии, поведенческие показатели.

**Ссылочный профиль:**
- Выгрузите внешние ссылки через Ahrefs, Serpstat или Google Search Console → Ссылки. Зафиксируйте, на какие URL они ведут.

## Шаг 2. Составьте карту редиректов

Это самый важный документ при любом обновлении. Формат простой:

```
Старый URL → Новый URL → Тип редиректа (301)
```

**Правила:**
- Каждый старый URL должен иметь соответствие. Нет исключений.
- Используйте 301 (постоянный) редирект. 302 не передаёт ссылочный вес.
- Не делайте цепочек редиректов (A → B → C). Только прямые.
- Если страница удаляется без аналога — редирект на ближайший по теме раздел.

Если URL-структура не меняется (что идеально) — карта редиректов не нужна. Именно поэтому я рекомендую при [редизайне](/services/redizajn-sajta/) сохранять существующие адреса, когда это возможно.

## Шаг 3. Сохраните мета-теги и микроразметку

Перед запуском нового дизайна убедитесь:

- **Title и description** перенесены для каждой страницы. Не подменены шаблонными.
- **Canonical** указывает на правильный URL (не на dev-версию).
- **Open Graph и Twitter Cards** заполнены — иначе при шаринге в соцсетях будет пустота.
- **Schema.org разметка** сохранена: FAQ, Organization, BreadcrumbList, Article.
- **Hreflang** (если сайт мультиязычный) — без ошибок.

Частая ошибка: разработчик делает новый шаблон и забывает блок с мета-тегами. На dev-сервере это незаметно, а после запуска — катастрофа.

## Шаг 4. Сохраните контент и внутреннюю перелинковку

Контент — основа тематической релевантности. Поисковые системы оценивают, какие сущности раскрыты на странице, насколько полно покрыты интенты пользователя. Если при редизайне текст урезали или переместили — релевантность падает.

**Что проверить:**
- Весь текстовый контент на месте. Ни один абзац не потерялся при переносе.
- Заголовки H2–H6 сохранили иерархию. Один H1 на страницу.
- Внутренние ссылки работают. Если URL изменились — ссылки обновлены.
- Изображения с alt-атрибутами перенесены.

Внутренняя перелинковка — отдельная тема. Ссылки между страницами формируют тематические связи, которые поисковики учитывают при оценке авторитетности темы. Разрыв этих связей ослабляет весь «кокон» контента.

## Шаг 5. Проверьте техническую часть

**Robots.txt:**
- Убедитесь, что новая версия не блокирует важные разделы. Частая ошибка — оставить `Disallow: /` после переезда с dev-сервера.

**Sitemap.xml:**
- Обновите карту сайта. Все новые URL на месте, старые — убраны.
- Отправьте обновлённый sitemap в Яндекс.Вебмастер и Google Search Console.

**Скорость:**
- Проверьте Core Web Vitals на новом дизайне до запуска. LCP < 2.5с, CLS < 0.1, INP < 200мс.
- Оптимизируйте изображения (WebP, lazy loading).
- Минифицируйте CSS/JS.

**SSL:**
- Проверьте, что HTTPS работает корректно на всех страницах.

**404:**
- Пройдитесь по сайту краулером после запуска. Ни одна страница не должна возвращать 404, если у неё есть входящие ссылки или трафик.

## Шаг 6. Запуск и мониторинг

**В день запуска:**
- Проверьте все редиректы (инструмент: httpstatus.io или curl).
- Убедитесь, что robots.txt открыт для индексации.
- Отправьте sitemap в поисковые системы.
- Проверьте 10–20 ключевых страниц вручную: мета-теги, контент, ссылки.

**Первая неделя:**
- Мониторьте индексацию в Search Console и Вебмастере. Ошибки краулинга, покрытие, рост 404.
- Следите за позициями по ключевым запросам.
- Проверяйте поведенческие показатели в Метрике: отказы, глубина, время на сайте.

**Первый месяц:**
- Сравните трафик с benchmark'ом из Шага 1.
- Если есть просадка — анализируйте, какие страницы потеряли позиции и почему.
- Проверьте, все ли страницы проиндексированы в новой версии.

## Чек-лист: SEO при редизайне

Скопируйте и используйте:

- [ ] Выгружен полный список URL с мета-тегами
- [ ] Зафиксированы позиции и трафик (benchmark)
- [ ] Составлена карта 301-редиректов
- [ ] Мета-теги перенесены на новый шаблон
- [ ] Микроразметка Schema.org сохранена
- [ ] Контент полностью перенесён
- [ ] Внутренние ссылки обновлены
- [ ] Robots.txt открыт для индексации
- [ ] Sitemap.xml обновлён и отправлен
- [ ] Core Web Vitals в зелёной зоне
- [ ] Нет битых ссылок (404)
- [ ] Мониторинг первые 2 недели после запуска

## Что делать, если позиции всё-таки просели

Не паникуйте. Небольшая турбулентность в первые 1–2 недели после обновления — нормально. Поисковые системы переиндексируют страницы и пересчитывают показатели.

Если просадка длится больше 3 недель:
1. Проверьте редиректы — нет ли 404 или цепочек.
2. Сравните мета-теги старых и новых страниц.
3. Убедитесь, что контент не урезан.
4. Проверьте скорость — возможно, новый дизайн медленнее.
5. Обратитесь к специалисту по [SEO-продвижению](/services/seo-prodvizhenie/) для детального аудита.
