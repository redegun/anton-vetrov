ИИ-агенты умеют писать код, отвечать на вопросы и анализировать данные. Но попросите Claude Code отрендерить сцену в Blender или обработать фото в GIMP — и начнётся боль. Агент не может кликать по кнопкам, у большинства программ нет API, а автоматизация через скриншоты — хрупкая и медленная.

Проект **CLI-Anything** от исследовательской группы HKUDS (Гонконгский университет) решает эту проблему радикально: он автоматически генерирует полноценный CLI-интерфейс для любого приложения. Одной командой.

## Проблема: софт создан для людей, а не для агентов

Сейчас ИИ-агенты работают с программами тремя способами — и все три так себе:

**Автоматизация GUI** — агент делает скриншоты, ищет кнопки, кликает. Медленно, ломается при любом обновлении интерфейса, жрёт ресурсы на распознавание картинок.

**Обёртки над API** — работает, но API есть у 5% софта. У GIMP, Blender, LibreOffice — только внутренние скриптовые языки с кривой документацией.

**Урезанные реимплементации** — агент пишет свой код для простых операций. Теряется 90% функциональности оригинала.

CLI-Anything предлагает четвёртый путь: взять исходный код приложения и сгенерировать из него настоящий CLI, который вызывает реальные функции программы.

## Как это работает

CLI-Anything — плагин для Claude Code (и других ИИ-платформ). Запускаете одну команду:

```
/cli-anything:cli-anything ./gimp
```

Дальше автоматический пайплайн из 7 фаз:

1. **Анализ** — сканирует исходный код, находит все функции и связи между ними
2. **Проектирование** — создаёт архитектуру CLI: группы команд, модель состояний, форматы вывода
3. **Реализация** — генерирует Python CLI на базе Click с REPL-интерфейсом, JSON-выводом, undo/redo
4. **Планирование тестов** — составляет план unit- и e2e-тестов
5. **Написание тестов** — реализует тестовый набор
6. **Документация** — создаёт TEST.md и SKILL.md (чтобы другие агенты знали, как пользоваться)
7. **Публикация** — генерирует setup.py, устанавливает CLI в PATH

На выходе — обычная Python-утилита, которую можно вызывать из терминала:

```bash
cli-anything-gimp project new --width 1920 --height 1080
cli-anything-gimp layer add -n "Background" --type solid --color "#1a1a2e"
cli-anything-blender render scene --engine cycles --samples 128 -o render.png
```

Агент получает структурированный JSON в ответ. Никакого парсинга скриншотов — чистые данные.

## CLI-Hub: магазин готовых CLI

Не хотите генерировать самостоятельно — идите в CLI-Hub. Это каталог готовых CLI от сообщества. Установка одной командой:


Есть мета-скилл: агент сам заходит в каталог, находит нужный CLI под задачу, устанавливает и начинает работать. Без участия человека.

## Почему CLI, а не MCP или API?

Справедливый вопрос. MCP (Model Context Protocol) — тоже способ подключить агента к внешним инструментам. Но у CLI есть преимущества:

**Универсальность** — CLI работает на любой ОС, не требует отдельного сервера, не зависит от конкретного провайдера.

**Самодокументирование** — флаг `--help` автоматически описывает все команды. Агент может разобраться сам.

**Композиция** — команды цепочатся через пайпы, как в классическом Unix. Сложные workflow собираются из простых кирпичиков.

**Надёжность** — текстовый ввод-вывод детерминирован. Нет хрупкости GUI-автоматизации и нет зависимости от внешних API.

**Проверенный подход** — Claude Code и так работает через CLI ежедневно. Тысячи рабочих сценариев каждый день.

## Что это значит на практике

Несколько сценариев, которые раньше были сложными, а теперь тривиальными:

**Пакетная обработка фото.** «Возьми 200 фото из папки, обрежь до 1200×628, добавь водяной знак, сохрани в WebP.» Агент использует cli-anything-gimp и обрабатывает всё за минуты.

**3D-рендеринг по описанию.** «Создай сцену: стол, на столе ноутбук, мягкий свет слева, рендер 4K.» Агент собирает сцену в Blender через cli-anything-blender и рендерит.

**Автоматизация документов.** «Сгенерируй 50 договоров из шаблона, подставив данные из CSV.» LibreOffice через CLI — без макросов и VBA.

**Видеомонтаж.** «Нарежь вебинар на главы по таймкодам, добавь интро и субтитры.» Shotcut или FFmpeg через CLI.

**Аудит электронных схем.** Инженер загружает проект KiCad и просит агента проверить DRC (Design Rule Check), найти незаземлённые цепи и экспортировать gerber-файлы для производства. Раньше — ручная проверка в GUI. Теперь — `cli-anything-kicad drc run && cli-anything-kicad export gerber`.

**Генерация отчётов из данных.** «Возьми CSV с продажами за квартал, построй 5 графиков, собери PDF-отчёт на 10 страниц с выводами.» Агент берёт LibreOffice Calc для графиков и Writer для вёрстки отчёта. Всё через CLI, никаких макросов.

**Автоматизация стримов.** Стример просит: «Настрой OBS — 3 сцены, вебкамера с хромакеем, оверлей с чатом, запись в MKV с резервной копией.» Агент через cli-anything-obs создаёт сцены, добавляет источники, настраивает кодировщик. Без единого клика в интерфейсе.

**Архитектурная визуализация.** «Возьми чертёж из FreeCAD, выдави стены, добавь окна из библиотеки, отрендери фасад в дневном и вечернем освещении.» FreeCAD + Blender через CLI-цепочку — один агент собирает модель, другой рендерит.

**Пакетная обработка научных публикаций.** Исследователь: «У меня 300 PDF статей в Zotero. Экспортируй библиографию в BibTeX, отфильтруй по ключевым словам, собери обзорную таблицу.» cli-anything-zotero делает это за минуты вместо часов ручной работы.

**Создание диаграмм из кода.** «Прочитай мой Python-проект, построй диаграмму классов и sequence-диаграмму основного flow в Draw.io.» Агент анализирует код, генерирует XML для Draw.io через CLI, экспортирует в PNG для документации.

Каждый из этих сценариев раньше требовал ручной работы, кастомных скриптов или глубокого знания конкретной программы. Теперь агент справляется сам — достаточно описать задачу на человеческом языке.

## Ограничения

Не всё идеально. Есть нюансы:

- Нужен **исходный код** приложения. Закрытый софт (Photoshop, Figma) напрямую не поддерживается — но можно использовать open-source аналоги.
- Генерация CLI — это **задача для мощной модели**. Claude Opus или GPT-5 справляются, модели послабее могут не вытянуть анализ крупной кодовой базы.
- Некоторые приложения требуют запущенного GUI-процесса в фоне. CLI управляет ими через IPC, но запустить окно всё равно нужно.
- Проект молодой — первый релиз в марте 2026. Будут баги и edge cases.

## Почему это важно

CLI-Anything — не просто инструмент. Это сдвиг парадигмы.

До сих пор мы адаптировали ИИ к существующему софту: писали обёртки, плагины, интеграции. CLI-Anything переворачивает логику: теперь софт адаптируется к агентам. Любой софт, автоматически.

Если тренд сохранится, через пару лет «агент-нативность» станет таким же ожидаемым свойством программы, как мобильная версия или API. CLI-Anything показывает, как туда прийти без переписывания всего с нуля.

## Как попробовать

1. Установите Claude Code (или другой поддерживаемый агент)
2. Добавьте плагин: `/plugin marketplace add HKUDS/CLI-Anything`
3. Установите: `/plugin install cli-anything`
4. Направьте на любую папку с исходным кодом: `/cli-anything:cli-anything ./your-software`

Исходный код проекта: [github.com/HKUDS/CLI-Anything](https://github.com/HKUDS/CLI-Anything)

Каталог готовых CLI: [CLI-Hub](https://hkuds.github.io/CLI-Anything/)

Проект распространяется под MIT-лицензией. Можно использовать в коммерческих проектах.
