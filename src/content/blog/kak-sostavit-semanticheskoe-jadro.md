Семантическое ядро — это полный список ключевых слов, по которым ваш сайт будет ранжироваться в поисковиках. Это не просто набор запросов — это структурированная карта, определяющая архитектуру сайта и контент-план.

Без семантического ядра SEO-продвижение превращается в хаотичную публикацию статей наугад.

## Что такое семантическое ядро

Семантическое ядро = все ключевые запросы вашей ниши, сгруппированные по смыслу и привязанные к конкретным страницам сайта.

**Из чего состоит:**

| Компонент | Пример |
|-----------|--------|
| Ключевой запрос | seo оптимизация сайта |
| Частотность | 281/мес (Яндекс Wordstat) |
| Интент | Информационный |
| Кластер | SEO-оптимизация |
| Целевая страница | /blog/seo-optimizacija-sajta/ |
| Конкуренция | Средняя |

**Размер ядра** зависит от ниши:
- Блог → 100–500 запросов
- Интернет-магазин → 1,000–50,000 запросов
- Сервисный сайт → 200–2,000 запросов

## Шаг 1: Определите базовые запросы

Начните с 10–20 «маркерных» запросов — основных тем вашего сайта.

**Источники маркерных запросов:**
- Ваши услуги / товары
- Проблемы вашей аудитории
- Вопросы, которые задают клиенты
- Категории на сайтах конкурентов
- Тематические форумы и сообщества

**Пример для SEO-блога:**
```
seo оптимизация
продвижение сайта
ключевые слова
мета-теги
ссылочное продвижение
семантическое ядро
seo аудит
```

## Шаг 2: Расширьте ядро

Каждый маркерный запрос расширяете через инструменты:

### Яндекс.Wordstat

Основной инструмент для русскоязычного SEO:

1. Введите маркерный запрос
2. Соберите все варианты из левой колонки
3. Проверьте правую колонку (ассоциации)
4. Повторите для каждого маркерного запроса

**Совет:** используйте расширение «Wordstat Assistant» для Chrome — позволяет быстро копировать запросы.

### Подсказки поисковиков

Яндекс и Google подсказывают популярные запросы:

- Начните вводить запрос → смотрите подсказки
- Блок «Вместе с этим ищут» внизу выдачи
- Блок «Люди также спрашивают» (Google)

### Конкуренты

Проанализируйте 3–5 конкурентов через:

- **Keys.so** — видимость домена в Яндексе
- **Serpstat** — ключевые слова и позиции
- **Ahrefs** — полный анализ органики

Выгрузите запросы конкурентов и добавьте те, которых нет в вашем ядре.

### Дополнительные источники

- Google Keyword Planner (через Google Ads)
- Answer the Public (вопросы по теме)
- AlsoAsked.com (связанные вопросы)
- Форумы, Reddit, Ответы.Mail.ru — как люди формулируют проблемы

## Шаг 3: Очистите ядро

После сбора в ядре будут тысячи запросов. Нужно почистить:

**Удалите:**
- Нерелевантные запросы (не ваша тема)
- Нулевые запросы (частотность в кавычках = 0)
- Дубли и синонимы (оставьте один вариант)
- Запросы с чужим брендом (если не пишете обзор)
- Навигационные запросы конкурентов

**Проверьте частотность в кавычках:**

Wordstat без кавычек показывает суммарную частоту всех вариаций. Проверяйте в формате `"!ключевое !слово"` — это реальная частота.

| Запрос | Без кавычек | В кавычках с ! |
|--------|------------|----------------|
| seo оптимизация | 5,000 | 1,138 |
| сео продвижение | 3,000 | 860 |
| как раскрутить сайт | 2,000 | 150 |

## Шаг 4: Кластеризация

Кластеризация — группировка запросов, которые можно продвигать на одной странице.

### Принцип кластеризации

Два запроса попадают в один кластер, если по ним в топ-10 выдачи попадают одинаковые URL. Это значит, что поисковик считает эти запросы одной темой.

### Методы кластеризации

**Ручная (до 300 запросов):**
- Группируйте по смыслу
- Проверяйте выдачу — если одни и те же сайты в топе, запросы в одном кластере

**Автоматическая (300+ запросов):**
- Just-Magic — лучший для Яндекса
- Rush Analytics — быстрый и точный
- Serpstat — встроенная кластеризация
- KeyAssort — десктопное приложение

### Пример кластера

```
Кластер: «Мета-теги»
├── meta теги сео (120)
├── title description h1 (45)
├── как правильно писать title (30)
├── мета описание сайта (25)
├── как заполнить description (15)
└── → Целевая страница: /blog/meta-tegi-title-description-h1/
```

## Шаг 5: Определите тип страниц

Для каждого кластера определите тип страницы:

| Тип интента | Тип страницы | Пример |
|-------------|-------------|--------|
| Информационный | Статья в блоге | «Как составить семантическое ядро» |
| Коммерческий | Страница услуги | «SEO-продвижение сайта» |
| Транзакционный | Карточка товара / Заказ | «Заказать SEO-аудит» |
| Навигационный | Категория / Раздел | «Блог об SEO» |

## Шаг 6: Постройте структуру сайта

На основе кластеров стройте архитектуру:

```
Главная
├── Услуги
│   ├── SEO-продвижение (кластер 1)
│   ├── Контекстная реклама (кластер 2)
│   └── Разработка сайтов (кластер 3)
├── Блог
│   ├── SEO-оптимизация сайта (кластер 4)
│   ├── Как подобрать ключевые слова (кластер 5)
│   ├── Мета-теги (кластер 6)
│   └── ...
└── О компании
```

**Правила:**
- Один кластер = одна страница
- Не размазывайте один запрос по нескольким страницам (каннибализация)
- Важные страницы — ближе к главной (меньше кликов)
- Связывайте тематически близкие страницы перелинковкой

## Шаг 7: Приоритизация

Не все кластеры одинаково важны. Приоритизируйте:

| Приоритет | Критерии |
|-----------|---------|
| Высокий | Высокая частотность + средняя конкуренция + коммерческий интент |
| Средний | Средняя частотность + информационный интент |
| Низкий | Низкая частотность + высокая конкуренция |

**Для нового сайта:** начинайте с НЧ-кластеров (низкая конкуренция) → набирайте авторитет → переходите к СЧ и ВЧ.

## Ошибки при составлении ядра

1. **Собрали и забыли** — ядро нужно обновлять минимум раз в квартал
2. **Только ВЧ-запросы** — конкуренция слишком высока для нового сайта
3. **Нет кластеризации** — 5 статей на одну тему каннибализируют друг друга
4. **Игнорирование интента** — коммерческий запрос на информационную страницу не даст конверсий
5. **Слишком мало запросов** — 20 ключей — это не ядро, а список пожеланий

## Заключение

Семантическое ядро — это фундамент SEO. Без него вы не понимаете, какой контент создавать и какие страницы оптимизировать.

Процесс: маркерные запросы → расширение через Wordstat и конкурентов → чистка → кластеризация → распределение по страницам → приоритизация.
